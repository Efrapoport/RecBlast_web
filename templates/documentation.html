Learn how to use RecBlast web

Documentation



RecBlast web takes several parameters as input:

1. Reference taxon: The reference taxon can be inserted as text (e.g. Homo sapiens) or as taxon number (e.g. 9606).

Homo Sapiens

2. List of target taxa: the web server limits the amount of taxa to 10. The taxa list can be inserted as a list of names, taxa ids, or a mix of both separated by a new line. The data can be inserted manually, or uploaded as a file. An example for a correct structure:

9606
Hydra Vulgaris
5202

3. List of biological units of interests: the user can upload a file/type names of genes, Uniprot ids, accession numbers, gis or a a mix of both separated by a new line. An example for a correct structure:

ADCY1
Q08828
XP_011291944.1

4. Run Name: a name given to the specific run.

5. BLAST parameters: E-Value (0-10), Coverage percentage (0-100) and Identity percentage (0-100).
An optional backwards e-value can be supplied and used for the second BLAST back against the reference taxon.

6. String Similarity: A number between 0 to 1 representing the similarity between two given genes.
A result is determined identical if its protein sequence is similar in more than 99% to the canonical protein sequence of the inspected gene, or if the similarity of the annotation of both proteins is than the 'string similarity' parameter defined by the user.
